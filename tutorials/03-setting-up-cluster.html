üéØ Choosing Your Kubernetes Environment

Before diving into Kubernetes, you need a cluster to work with. There are two main approaches:

  • Local Development: Run Kubernetes on your laptop (perfect for learning)
  • Cloud Managed: Use a cloud provider's managed service (production-ready)

üíª Local Kubernetes Options

üöÄ minikube

Best for: Learning and development

Pros:

  • Easy to install
  • Single-node cluster
  • Supports multiple drivers
  • Built-in addons
  • Cross-platform

Cons:

  • Single node only
  • Resource intensive

üê≥ Docker Desktop

Best for: Quick start on Mac/Windows

Pros:

  • One-click enable
  • Integrated with Docker
  • Simple UI
  • Good for beginners

Cons:

  • Limited configuration
  • Single node
  • Requires Docker Desktop

üé™ kind

Best for: CI/CD and testing

Pros:

  • Multi-node clusters
  • Fast startup
  • Runs in Docker
  • Great for testing

Cons:

  • Command-line only
  • Less beginner-friendly

‚úÖ Recommendation

For this tutorial, we'll use minikube because it's:

  • Easy to install on all platforms
  • Feature-rich with addons
  • Perfect for learning
  • Well-documented

üöÄ Installing minikube

Prerequisites

  • 2 CPUs or more
  • 2GB of free memory
  • 20GB of free disk space
  • Internet connection
  • Container or virtual machine manager (Docker, VirtualBox, etc.)

Installation by Platform

üêß Linux

# Download minikube
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64

# Install minikube
sudo install minikube-linux-amd64 /usr/local/bin/minikube

# Verify installation
minikube version

üçé macOS

# Using Homebrew (recommended)
brew install minikube

# Or download directly
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-darwin-amd64
sudo install minikube-darwin-amd64 /usr/local/bin/minikube

# Verify installation
minikube version

ü™ü Windows

# Using Chocolatey
choco install minikube

# Or using winget
winget install Kubernetes.minikube

# Or download installer from:
# https://minikube.sigs.k8s.io/docs/start/

# Verify installation (PowerShell)
minikube version

Installing kubectl

kubectl is the command-line tool for interacting with Kubernetes clusters.

üêß Linux

curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
kubectl version --client

üçé macOS

brew install kubectl
kubectl version --client

ü™ü Windows

choco install kubernetes-cli
kubectl version --client

üéÆ Essential kubectl Commands

Cluster Information

# Get cluster info
kubectl cluster-info

# Get cluster version
kubectl version

# View cluster configuration
kubectl config view

# Get current context
kubectl config current-context

Resource Management

# Get resources
kubectl get nodes
kubectl get pods
kubectl get services
kubectl get deployments

# Get all resources
kubectl get all

# Get resources in all namespaces
kubectl get pods --all-namespaces
# or
kubectl get pods -A

# Describe a resource (detailed info)
kubectl describe node minikube
kubectl describe pod

Creating Resources

# Create from file
kubectl apply -f deployment.yaml

# Create from URL
kubectl apply -f https://example.com/deployment.yaml

# Create resource imperatively
kubectl create deployment nginx --image=nginx

# Delete resource
kubectl delete deployment nginx

Debugging

# View logs
kubectl logs 

# Follow logs (like tail -f)
kubectl logs -f 

# Execute command in pod
kubectl exec -it  -- /bin/bash

# Port forward to local machine
kubectl port-forward  8080:80

üéØ Hands-On: Deploy Your First Application

Step 1: Create a Deployment

kubectl create deployment hello-minikube --image=kicbase/echo-server:1.0

# Output:
# deployment.apps/hello-minikube created

Step 2: Check Deployment Status

kubectl get deployments

# Output:
# NAME             READY   UP-TO-DATE   AVAILABLE   AGE
# hello-minikube   1/1     1            1           30s

kubectl get pods

# Output:
# NAME                              READY   STATUS    RESTARTS   AGE
# hello-minikube-5d9b964bfb-xxxxx   1/1     Running   0          30s

Step 3: Expose as a Service

kubectl expose deployment hello-minikube --type=NodePort --port=8080

# Output:
# service/hello-minikube exposed

kubectl get services

# Output:
# NAME             TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)          AGE
# hello-minikube   NodePort    10.96.123.45            8080:30123/TCP   10s
# kubernetes       ClusterIP   10.96.0.1               443/TCP          10m

Step 4: Access the Application

# Get the URL
minikube service hello-minikube --url

# Output:
# http://192.168.49.2:30123

# Or open in browser automatically
minikube service hello-minikube
üéâ Congratulations! You've deployed your first application to Kubernetes!

Step 5: Scale the Application

# Scale to 3 replicas
kubectl scale deployment hello-minikube --replicas=3

# Check the pods
kubectl get pods

# Output:
# NAME                              READY   STATUS    RESTARTS   AGE
# hello-minikube-5d9b964bfb-xxxxx   1/1     Running   0          5m
# hello-minikube-5d9b964bfb-yyyyy   1/1     Running   0          10s
# hello-minikube-5d9b964bfb-zzzzz   1/1     Running   0          10s

Step 6: Clean Up

# Delete the service
kubectl delete service hello-minikube

# Delete the deployment
kubectl delete deployment hello-minikube

# Verify deletion
kubectl get all

üîß Useful minikube Commands

Cluster Management

# Start cluster
minikube start

# Stop cluster (preserves state)
minikube stop

# Delete cluster
minikube delete

# Pause cluster (saves resources)
minikube pause

# Unpause cluster
minikube unpause

# Get cluster status
minikube status

Addons

# List available addons
minikube addons list

# Enable an addon
minikube addons enable dashboard
minikube addons enable metrics-server
minikube addons enable ingress

# Disable an addon
minikube addons disable dashboard

# Open Kubernetes dashboard
minikube dashboard

Advanced Options

# Start with specific resources
minikube start --cpus=4 --memory=8192

# Start with specific Kubernetes version
minikube start --kubernetes-version=v1.28.0

# Start with specific driver
minikube start --driver=docker
minikube start --driver=virtualbox

# SSH into minikube node
minikube ssh

# Get minikube IP
minikube ip

üêõ Troubleshooting Common Issues

‚ùå Issue: minikube start fails

Solution:

# Delete and restart
minikube delete
minikube start

# Try different driver
minikube start --driver=docker

# Check logs
minikube logs

‚ùå Issue: kubectl not connecting

Solution:

# Update kubectl context
minikube update-context

# Verify context
kubectl config current-context

# Should output: minikube

‚ùå Issue: Not enough resources

Solution:

# Start with minimal resources
minikube start --cpus=2 --memory=2048

# Or increase Docker resources in Docker Desktop settings

üéì kubectl Cheat Sheet

Quick Reference

# Get resources
kubectl get pods                    # List pods
kubectl get svc                     # List services
kubectl get deploy                  # List deployments
kubectl get all                     # List all resources

# Describe resources
kubectl describe pod          # Detailed pod info
kubectl describe node         # Detailed node info

# Create/Update resources
kubectl apply -f file.yaml          # Create/update from file
kubectl create deployment     # Create deployment
kubectl expose deployment     # Create service

# Delete resources
kubectl delete pod            # Delete pod
kubectl delete -f file.yaml         # Delete from file

# Logs and debugging
kubectl logs              # View logs
kubectl logs -f           # Follow logs
kubectl exec -it  -- bash      # Shell into pod

# Scaling
kubectl scale deploy  --replicas=3

# Port forwarding
kubectl port-forward  8080:80

# Get YAML/JSON
kubectl get pod  -o yaml      # Get pod YAML
kubectl get pod  -o json      # Get pod JSON

# Shortcuts
kubectl get po                      # pods
kubectl get svc                     # services
kubectl get deploy                  # deployments
kubectl get ns                      # namespaces

üåê Next Steps: Cloud Kubernetes (Optional)

Once you're comfortable with minikube, you can explore cloud-managed Kubernetes:

AWS EKS

eksctl create cluster \
  --name my-cluster \
  --region us-west-2 \
  --nodegroup-name standard \
  --node-type t3.medium \
  --nodes 3

Google GKE

gcloud container clusters \
  create my-cluster \
  --zone us-central1-a \
  --num-nodes 3 \
  --machine-type n1-standard-2

Azure AKS

az aks create \
  --resource-group myRG \
  --name myCluster \
  --node-count 3 \
  --node-vm-size Standard_DS2_v2
